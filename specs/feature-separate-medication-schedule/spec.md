# Feature Spec: Medication と Schedule の分離

## 1. 目的
薬マスタと服薬スケジュールを分離し、ユーザーが「いつ何をまとめて飲むか」をスケジュール単位で管理できるようにする。

## 2. スコープ
### 2.1 対象
- 一覧/登録の主語を「薬」から「スケジュール」へ変更
- 薬マスタ管理画面の独立
- スケジュールと薬のN:N関連付け
- 通知/履歴/ウィジェットのスケジュール中心化
- 既存データの移行

### 2.2 非対象
- クラウド同期
- 認証/アカウント管理
- 医療機関連携

## 3. ユースケース
### UC-1: スケジュールを作成する
- 事前条件: 薬マスタが1件以上存在する
- 入力: `name`, `slot`, `timeMinutes`, `medicationIds[]`
- 正常系:
  1. ユーザーが必須項目を入力
  2. 保存実行
  3. スケジュールと関連を保存
  4. 通知を再構成
- 例外:
  - 薬未選択: 保存不可
  - 名称空: 保存不可

### UC-2: スケジュールを編集する
- 事前条件: 対象スケジュールが存在
- 入力: UC-1と同じ
- 正常系: 更新後に通知・ウィジェットを更新
- 例外: 競合/不存在時はエラー表示し一覧へ復帰

### UC-3: スケジュールを実行（飲んだ）する
- 事前条件: スケジュールが有効、関連薬が1件以上
- 入力: `scheduleId`, 実行時刻（通常は現在時刻）
- 正常系:
  1. 同一 `scheduleId` + `takenAt` の重複を確認
  2. 関連薬ごとに履歴を同一時刻で追加
  3. 5分抑止を設定
  4. 履歴/ウィジェットを更新
- 例外:
  - 重複: 保存せず通知
  - 関連薬0件: 実行不可

### UC-4: 履歴を手動追加する
- 事前条件: スケジュールが存在
- 入力: `scheduleId`, `takenAt`
- 正常系: 対象スケジュールに紐づく薬履歴を一括追加
- 例外:
  - 未来日時: 保存不可
  - 重複: 重複分はスキップ

### UC-5: 薬マスタを管理する
- 正常系: 薬のCRUD
- ルール: スケジュールに紐づく薬を削除する場合は確認し、参照整合を保つ

### UC-6: 通知を受け取り一覧へ戻る
- 正常系:
  1. 指定時刻で通知表示
  2. タップで一覧画面を開く
  3. 次回通知を再構成

### UC-7: ウィジェットからスケジュール実行する
- 正常系: 一覧ウィジェット上の実行操作でUC-3相当の記録を行う

## 4. 入出力仕様
### 4.1 スケジュール登録入力
- `name`: 1..64文字
- `slot`: `MORNING` / `NOON` / `EVENING`
- `timeMinutes`: 0..1439
- `medicationIds`: 1件以上

### 4.2 履歴出力
- 一覧表示単位: `scheduleId + takenAt` を1グループ
- 詳細表示: グループ内の薬履歴行
- 表示項目: スケジュール名、日時、薬件数、誤登録状態

## 5. 業務ルール
- BR-1: スケジュール保存時に薬選択は必須
- BR-2: 同一スケジュールの同一日時記録は重複禁止
- BR-3: 手動追加で未来日時を禁止
- BR-4: スケジュール実行時、対象薬は全件同一 `takenAt`
- BR-5: 実行後5分間は同一スケジュールの再実行を無効化
- BR-6: スケジュール0件の場合は通知を全停止

## 6. バリデーション
- 名称: 空文字不可、前後空白除去後で判定
- 時刻: 0..1439以外は不可
- 薬選択: 0件不可
- 手動追加日時: 現在時刻より未来不可

## 7. 状態変化
- `Schedule.active = true/false` に応じて通知対象を切替
- 履歴の `incorrectAt` は null/非null をトグル
- 5分抑止は `lastTakenAt + 5min` を超過で自動解除扱い

## 8. 通知仕様
- スケジュール時刻ごとにアラームを生成
- 受信時に該当スケジュールの有効性を再評価
- 変更/削除時は即時再構成
- 端末再起動時に全件再構成

## 9. ウィジェット仕様
- スケジュールウィジェット
  - 行表示: スケジュール名、時刻、薬サマリ
  - 操作: 実行ボタン
- 履歴ウィジェット
  - 直近グループ履歴を表示
  - タップで履歴画面へ遷移

## 10. データ移行仕様
- MG-1: 既存 `medications` は薬マスタとして保持
- MG-2: 既存薬の時間帯/時刻から初期スケジュールを生成
- MG-3: 初期スケジュールに当該薬を1件紐づけ
- MG-4: 既存履歴は `scheduleId = null` で保持し、可能なら補完
- MG-5: 移行失敗時はロールバックし、既存データを維持

## 11. 完了条件との対応
本仕様の完了条件は `acceptance.md` のAC群で判定する。
