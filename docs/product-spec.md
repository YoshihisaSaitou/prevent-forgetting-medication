# Product Specification

## 1. プロダクト概要
本プロダクトは、服薬忘れを防ぐためのAndroidアプリである。
`feature-separate-medication-schedule` では、服用対象（薬）と服用運用（スケジュール）を分離し、日常運用に近い「時間単位での管理」を中心に据える。

## 1.1 実装方針（ドメイン駆動設計）
- 本仕様はドメイン駆動設計（DDD）で実装する。
- モデルの中心は `Medication`（薬マスタ）と `Schedule`（服薬運用）とし、UI都合ではなくドメインルールを優先する。
- 実装時は Context 境界（Medication Catalog / Schedule Planning / Intake Tracking）を保ち、責務の混在を避ける。

## 2. ドメインモデル（To-Be）
- 薬マスタ（Medication）
  - 服用対象の定義（名称、食前/食後、メモ）
  - 時刻や通知責務は持たない
- スケジュール（Schedule）
  - 服薬の実行単位（スケジュール名、時間帯、時刻）
  - 複数薬を紐づける
- スケジュール薬関連（ScheduleMedication）
  - スケジュールと薬のN:N関連を保持
- 服薬履歴（IntakeHistory）
  - スケジュール実行時に、対象薬ごとの履歴行を同一時刻で記録
  - スケジュールIDを保持し、スケジュール単位表示を可能にする

## 3. 主要機能
### 3.1 スケジュール一覧
- 一覧行に以下を表示する
  - スケジュール名
  - 時間帯（朝/昼/夕）と時刻
  - 対象薬のサマリ（件数または先頭N件）
- 行アクション
  - 飲んだ（スケジュール実行）
  - 編集
  - 削除（確認ダイアログ必須）
- データ0件時は空状態メッセージを表示

### 3.2 スケジュール登録/編集
- 必須入力
  - スケジュール名
  - 時間帯
  - 時刻
  - 対象薬（1件以上）
- 任意入力
  - 補足メモ（将来拡張）
- バリデーションエラーは保存せず、入力項目を維持したまま明示表示

### 3.3 薬マスタ管理
- 薬の登録・編集・削除・一覧
- 薬自体は時刻を持たない
- 薬削除時、関連スケジュールからの参照整合を保証する

### 3.4 服薬記録（スケジュール実行）
- 1回の「飲んだ」で、紐づく薬すべてを同一 `takenAt` で記録する
- 同一スケジュール・同一日時の重複実行は拒否する
- 実行後5分は同一スケジュールの再実行を無効化する

### 3.5 履歴
- デフォルトはスケジュール単位表示
- 詳細表示で薬内訳を参照可能
- 手動追加は過去日時のみ許可（未来禁止）
- 誤登録フラグ（`incorrectAt`）の付与/解除を可能にする

### 3.6 通知
- スケジュール時刻に単発アラームを設定
- 発火時に該当スケジュールが有効であれば通知表示
- 通知タップでアプリの一覧画面へ遷移
- 端末再起動時に通知を再構築

### 3.7 ホームウィジェット
- スケジュールウィジェット
  - スケジュール名・時刻・対象薬サマリを表示
  - ウィジェット上で「飲んだ」実行
- 履歴ウィジェット
  - 直近履歴を表示
  - タップで履歴画面へ遷移

## 4. 画面横断の共通ルール
- フッターメニューは「一覧=スケジュール一覧」「登録=スケジュール登録」を指す
- 薬マスタ管理画面へは一覧または設定から遷移可能にする
- DB更新後は関連ウィジェットを即時更新する
- 時刻は内部的に「1日内分（minutes）」で管理する

## 5. 認可・セキュリティ方針
- ユーザー認証なし（単一端末前提）
- 外部通信なし（ローカル完結）
- Android 13+ の通知権限は実行時に要求

## 6. エラー方針
- 入力エラー: 保存拒否 + ユーザー向け文言表示
- 重複記録: 保存スキップ件数を通知
- 権限不足（通知）: 許可依頼を表示し、未許可時は機能制限を明示
- DB例外: アプリクラッシュを避け、可能な限りUIで復帰可能な状態を維持

## 7. 非機能要件
- Android 7.0 以上（minSdk=24）
- 端末再起動後の通知復元
- 通知時刻の高精度性（`setExactAndAllowWhileIdle` 優先）
- オフライン常時利用可能

## 8. 既知の制約
- 初期実装は単一アプリモジュールで構成
- 本機能内ではクラウド同期・共有機能は扱わない
- 既存履歴との互換のため、履歴表示ロジックに移行期間の分岐が必要




## 9. テスト実施必須ツール
本プロダクト仕様に基づく実装・検証では、以下のテストツールを必須で実施する。
- JUnit
- MockK
- kotlinx-coroutines-test
- AndroidX Test（core/runner/rules）
- Espresso
- Robolectric
- Hilt Testing
