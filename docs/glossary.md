# Glossary

| 用語 | 定義 | 補足 |
|---|---|---|
| 薬マスタ（Medication） | 服用対象の基本情報。名称、食事タイミング、メモを保持する。 | 時刻情報を持たない。 |
| スケジュール（Schedule） | 服薬実行の単位。時間帯・時刻・対象薬集合を持つ。 | 一覧・通知・ウィジェットの主語。 |
| 時間帯（IntakeSlot） | 朝/昼/夕の区分。 | `MORNING` / `NOON` / `EVENING` |
| 食事タイミング（MealTiming） | 服薬と食事の前後関係。 | `BEFORE_MEAL` / `AFTER_MEAL` / 未設定 |
| スケジュール実行 | スケジュールに紐づく薬を同一時刻で一括記録する操作。 | UI上の「飲んだ」。 |
| 服薬履歴（IntakeHistory） | いつ、どの薬を、どのスケジュール由来で記録したかを表すデータ。 | `takenAt`, `createdAt`, `incorrectAt` を保持。 |
| 手動記録 | 履歴画面で日時指定して追加した記録。 | 通常 `createdAt != takenAt`。 |
| 誤登録 | 履歴が誤りであることを示す論理状態。 | `incorrectAt != null`。 |
| 重複記録 | 同一対象に対する同一日時の再記録。 | 保存を拒否/スキップする。 |
| 5分抑止 | 記録直後、同一対象の再実行を一定時間無効化する制御。 | 現行は薬単位、To-Beはスケジュール単位へ拡張。 |
| 共通時刻設定 | 朝/昼/夕の既定時刻設定。 | SharedPreferences管理。 |
| ウィジェット投影 | ホーム画面向けに整形された一覧表示データ。 | RemoteViewsで描画。 |
| 通知再構成 | DB/設定変更や再起動後にアラームを組み直す処理。 | AlarmManagerを使用。 |
| 移行（Migration） | 既存スキーマ・データを新モデルへ変換する工程。 | 破壊的移行は原則禁止。 |
